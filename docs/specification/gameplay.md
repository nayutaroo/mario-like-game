# gameplay — ゲーム全体の流れ

## ゲームループ

1. **タイトル画面** … ゲーム開始 / 操作説明への遷移
2. **ステージ開始** … 残機・スコア表示後にプレイ開始
3. **プレイ中** … 横（または縦）スクロールでゴールを目指す
4. **ステージ結果** … クリア時はスコア加算、ミス時は残機を減らしてリスポーン
5. **ボス戦（最終面）** … HP を削り切るとクリア
6. **エンディング** … 最終ステージクリアで簡単なクリア画面を表示

## クリア条件

- **【確定】** 通常面はステージ最奥のゴールに到達するとクリア（モチーフは「光の輪」、`level-design.md` 参照）。
- **【確定】** ボス面はボスを撃破するとクリア（`mechanics.md` のボス戦フォーマット参照）。
- **【確定】** ステージ内に制限時間を設定し、0 になるとミス扱い（具体値は実装でチューニング）。

## ミス条件

- **【確定】** 通常状態の敵に接触する
- **【確定】** 穴に落下する（画面外下方向）
- **【確定】** 制限時間切れ
- **【仮】** トラップ（針・溶岩等）に接触する（採用面は実装段階で決定）

## スコア

- **【確定】** 敵を倒す / ベリー（コイン代替）を取る / 制限時間残量でスコアが加算される
- **【確定】** ハイスコアは `localStorage` に保存する（細部は実装段階で判断）

## 残機・コンティニュー

- **【確定】** 開始時 3 機。ミスで 1 機減少、0 になるとゲームオーバー
- **【確定】** ゲームオーバー後はタイトルに戻る
- **【確定】** 100 ベリーで 1 機追加。1UP 専用アイテム「金のヒナ」（`mechanics.md` / `assets.md` 参照）でも 1 機追加

## セーブ / ポーズ

- **【確定】** ステージ進行のセーブ機能は持たない（毎回タイトルからスタート）。スコープ縮小のため割り切る。
- **【確定】** ポーズ機能を持つ（`controls.md` 参照）。

## 未決事項

- ステージ間のワールドマップ表示の有無 → 実装段階で「直接遷移」か「簡易マップ」かを決める
- 難易度選択の有無 → 小規模スコープなので無しの方向、必要なら実装段階で再検討
