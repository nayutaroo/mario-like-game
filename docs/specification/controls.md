# controls — 入力デバイスとキー配置

## 対応デバイス

- **【確定】** PC キーボードを必須とする
- **【確定】** タッチ操作（スマホ／タブレット／キーボード非搭載のネイティブアプリ環境）に対応するため、画面上にオンスクリーンボタンを重ねて表示する
- **【確定】** ゲームパッドは初期スコープ外（将来検討）

## キー配置

主操作は WASD / 矢印キーの両対応とし、アクションキーは右手側に置く。

| 操作 | キー（主） | キー（代替） |
| --- | --- | --- |
| 左移動 | ← | A |
| 右移動 | → | D |
| しゃがむ / 下入力 | ↓ | S |
| 上入力（扉に入る等） | ↑ | W |
| ジャンプ | スペース | K |
| ダッシュ（押し続けで走り） | Shift | J |
| 種を撃つ（リンゴ状態時） | Z | — |
| ポーズ | P | Esc |
| タイトル決定 / ゲーム開始 | Enter | スペース |

- **【確定】** ジャンプは押し続けで滞空時間が伸びる（可変高度ジャンプ）
- **【確定】** ダッシュは押し続けて移動するとトップスピードに到達する方式
- **【確定】** 種発射ボタンはダッシュとは独立（Z）

## オンスクリーンボタン（タッチ対応）

- **【確定】** プレイ画面の四隅に半透明のボタンを配置する。キーボード操作と並行して動作する（仮想入力フラグとキー入力の OR）
- **【確定】** 配置:
  - 左下: ← / → の方向ボタン
  - 右下: ジャンプ（大）/ ダッシュ / タネ発射 ボタン
- **【確定】** ボタンは押下中フラグでキー入力相当の状態を保持。指（マウス）を離すと release 扱い
- **【確定】** タッチデバイス（`'ontouchstart' in window` または `navigator.maxTouchPoints > 0`）または `?touch=1` クエリ時に表示する。デスクトップキーボード環境ではデフォルト非表示
- **【確定】** プレイ中のみ表示。タイトル／ヘルプ／ポーズ／ゲームオーバーでは非表示
- **【未決】** マルチタッチ（移動＋ジャンプ同時など）の完全サポート → 初期実装はマウスイベント互換のシングルタッチ前提。必要に応じて pointer ID ベースで拡張

## 入力に関する設計上の注意

- **【確定】** ブラウザのデフォルト挙動（スペースでスクロール、矢印でスクロール等）が発火しないように、ゲームキャンバスにフォーカスがあるときはキー入力をキャンセルする
- **【確定】** キーリピートに依存せず、押下中フラグで連続移動を扱う
- **【確定】** 入力バッファ（先行入力）はジャンプのみ短時間（実装段階でチューニング）採用する
