# mechanics — プレイヤー挙動・敵・アイテム・物理ルール

具体的な数値（速度、重力など）は実装時にチューニングする前提のたたき台。

## 主人公

- **【確定】** 丸いヒヨコのキャラクター「ピヨ」
- **【確定】** ポップでカートゥーン調。マリオを直接想起させない独自デザインにする。
- **【確定】** プロト段階では矩形＋絵文字（🐤）で表現。向きが分かるよう、顔の前側にくちばし状の小マーカーを表示する

## プレイヤーの状態

- **【確定】** 「通常」「パワーアップ」の段階を持ち、ダメージで段階が下がる。
- **【確定】** 3 段階構成：

  | 段階 | 名称 | 取得方法 | できること | 被ダメージ時 |
  | --- | --- | --- | --- | --- |
  | 1 | ノーマル | 初期状態 | 走る・ジャンプ | 死亡 |
  | 2 | どんぐり | ドングリを取る | 体が一回り大きくなり 1 回耐える | ノーマルに戻る |
  | 3 | リンゴ | リンゴを取る | 種を飛ばして攻撃できる | どんぐりに戻る |

- **【確定】** 葉っぱを取ると一定時間「ふわふわ落下」状態になる（落下速度低下）。
- **【確定】** ノーマル状態で被ダメージ → 即時リスポーンではなく、短時間の死亡演出（垂直に跳ね上がりつつ赤点滅、約 0.9 秒）を経てから現在のリスポーン位置に戻る
- **【確定】** 死亡演出中は入力・敵接触判定を無効化する（追撃ダメージや誤操作を防ぐ）

## 移動

- **【確定】** 左右移動、ジャンプ、ダッシュ（走り）が可能
- **【確定】** 歩き / 走りで最高速度が変わる
- **【確定】** ジャンプは押し続けで滞空時間が伸びる（可変高度ジャンプ）
- **【確定】** 空中でも左右の操作でわずかに加速・減速できる
- **【確定】** ブロックの下からジャンプして頭で叩ける

## 攻撃

- **【確定】** 敵を上から踏みつけると倒せる
- **【確定】** 踏み判定はプレイヤーが敵より一定以上（実装値 12 px）上に居ることを条件とする。同じ高さで横方向にぶつかった場合（ダッシュ含む）はダメージ判定として扱い、踏みにはしない
- **【確定】** リンゴ状態のときのみ、種を発射して攻撃できる
- **【確定】** 種は現在向いている方向（直近の左右入力で決まる facing）に飛ぶ

## 敵キャラクター

すべて本作の固有名。役割（歩兵・甲羅・飛行）はマリオ系の典型に倣う。

- **【確定】** **イモムシ** … 直進、壁にぶつかると反転。踏みつけで即死（歩兵タイプ）
- **【確定】** **ダンゴムシ** … 踏むと丸まる。再度蹴ると弾丸として転がり、他の敵を巻き込む（甲羅タイプ）
  - 蹴り出した直後の短い猶予（実装値 0.12 秒）はプレイヤー側からのダメージ判定をスキップする。これは「丸まり状態を蹴る接触」と「転がり中のダメージ接触」が同じ接触セッションで両方発火するのを防ぐため
- **【確定】** **コバチ** … 一定の高さを飛行して直進する飛行敵
- **【確定】** **ボス（仮称: ヌシイモ）** … 1-5 最終面に登場。3 回踏みつけで撃破。
  - 体力: 3。1 回踏むごとに 1 減少
  - 攻撃パターン: 左右移動 → ジャンプ叩きつけ → 短い隙、を繰り返す
  - 弱点: 上方向からの踏みつけ。横からの接触は被ダメージ
  - 種（リンゴ状態）でもダメージを与えられるが、種 5 発で 1 ダメージ相当（実装段階で再調整可）

## アイテム

- **【確定】** **ドングリ** … プレイヤーを「どんぐり」状態にする
- **【確定】** **リンゴ** … プレイヤーを「リンゴ」状態にし、種を撃てるようにする
- **【確定】** **葉っぱ** … 一定時間ふわふわ落下できる
- **【確定】** **ベリー（木の実）** … スコア加算。100 個で 1 機追加（コイン代替）
- **【確定】** **きらきら花** … 一定時間無敵
- **【確定】** **金のヒナ** … 1UP 専用アイテム。隠しエリア / ボーナス面に配置
- **【確定】** 取得済みアイテム（出現済みのもの）は、プレイヤーが死亡してリスポーンしても再出現しない（消費型扱い）

## ? ブロック（マリオ風隠しブロック）

- **【確定】** ステージ中に固体ブロック（黄色＋"?" 表記）として配置される。プレイヤーは上に乗ることもでき、地形扱い
- **【確定】** プレイヤーが**下から頭で叩いた時のみ**中身が出現する（衝突方向で判定: 横や上接触では発動しない）
- **【確定】** 中身は次のいずれか:
  - 単体アイテム（ドングリ / リンゴ / 葉っぱ / ベリー / きらきら花 / 金のヒナ）
  - ボーナス面へのワープ（💫 マーク）
- **【確定】** 一度叩くと使用済みになり、灰色に変色して二度と発動しない（地形としては残る）
- **【確定】** 叩いた瞬間にブロックが軽く跳ねる短いフィードバック演出（約 8 フレーム / 6 px）

## 物理ルール

- **【確定】** 重力により落下する
- **【確定】** 軸別の AABB 当たり判定（横スクロール 2D の典型実装）
- **【確定】** タイルベース（グリッドに乗る）。基準タイルサイズは 32×32 px
- **【確定】** 段差・坂は初期スコープでは「四角い地形のみ」とし、坂は対応しない

## すり抜け足場（one-way / semi-solid platform）

- **【確定】** 通常の固体ブロックとは別系統の足場。**上から落下してきたとき (vel.y ≥ 0) のみ着地**し、下から / 横から接触したときはすり抜ける
- **【確定】** 着地条件: 「直前フレームの底辺 y が板の上 ≤ ssTop」 かつ 「現フレームで重なる pos.y ≥ ssTop」 かつ 「降下中 vel.y ≥ 0」 の 3 条件すべて満たすときのみ snap
- **【確定】** 見た目は固体ブロックと区別する（透明度・色・装飾絵文字）。バリアント:
  - `cloud` (☁️ 空ステージ向け)
  - `branch` (🪵 森ステージ向け)
  - `grass` (🌿 草原向け)
  - `rock` (🪨 洞窟向け)
- **【確定】** ステージデータでは `semiSolids?: SemiSolidDef[]` フィールドで配置（既存 `platforms` とは独立）
- **【確定】** 中間ポイント／ゴール／敵スポーンには影響しない。あくまで地形要素

## リスポーン挙動

- **【確定】** 残機が減って復帰するイベント（被ダメージ死・落下死・時間切れ）すべてで、敵キャラクターは初期配置にリセットされる
- **【確定】** アイテムは前項の通り消費型扱い。リスポーン時に再出現しない
- **【確定】** スコア・ベリー数・残機・取得済み中間ポイントは保持される
- **【確定】** ステージ内の中間ポイントを通過済みであれば、その地点から復帰する（未通過ならスタート地点）

## カメラ・スクロール

- **【確定】** プレイヤー追従のスクロール
- **【確定】** 通常面は横スクロールのみ。左方向にはスクロール巻き戻ししない
- **【確定】** 一部ステージでは**縦スクロール**を許容する（採用ステージは `level-design.md` 参照）

## 未決事項

- 水中ステージ等の特殊地形 → 初期スコープ外、必要なら次フェーズ
