# architecture — ゲーム実装アーキテクチャ

`docs/specification/` の確定事項を Kaboom.js 上にどう乗せるかをまとめる。

## 全体方針

- **【確定】** Kaboom.js のシーン（Scene）機構をそのまま使い、ゲームの大枠の状態遷移はシーン切替で表現する
- **【確定】** ゲームオブジェクトは Kaboom.js の **コンポーネント合成パターン**で構築する（`add([sprite(...), pos(...), area(...), body(...), "tag", ...])`）
- **【確定】** 共通ロジック（プレイヤー状態遷移・敵 AI・タイル定義など）は素の TypeScript モジュールとして `src/` 配下に切り出し、シーンからインポートする

## キャンバス・解像度

- **【確定】** 仕様の通りベース解像度 **1280×720 px**、16:9 を維持
- **【確定】** Kaboom.js の `kaplay({ width: 1280, height: 720, letterbox: true, pixelDensity: 1 })` 相当で初期化し、ブラウザサイズに応じてレターボックス表示
- **【確定】** ピクセルアート用に `crisp: true`（または `pixel-perfect` オプション）でアンチエイリアシング無効化
- **【確定】** 基準タイルサイズ **32×32 px**

## シーン構成

- **【確定】** 以下のシーンを定義する：

  | シーン名 | 役割 | 主な遷移先 |
  | --- | --- | --- |
  | `title` | タイトル画面 | → `play(stage="1-1")` |
  | `play` | 通常面 / ボス面プレイ中（ステージID引数） | → `pause` / → `gameover` / → `clear` / → `bonus` |
  | `pause` | ポーズ画面（`play` を一時停止表示） | → `play` / → `title` |
  | `bonus` | ボーナス面（木の実の部屋） | → `play(次の通常面)` |
  | `gameover` | ゲームオーバー演出 | → `title` |
  | `clear` | 最終クリア演出 | → `title` |

- **【確定】** `play` シーンは引数で**ステージ ID** を受け取り、対応するレベルデータを読み込む（`scene("play", (opt: { stage: StageId }) => { ... })`）

## エンティティ・コンポーネント設計

### プレイヤー

- **【確定】** 1 つのゲームオブジェクトに以下のコンポーネントを合成:
  - `sprite("piyo")` … 状態（ノーマル / どんぐり / リンゴ / ふわふわ）でアニメーション切替
  - `pos()` / `area()` / `body()` … 位置・当たり判定・物理
  - `health(1)` … 状態段階の管理に流用（または独自コンポーネントで「pyo-state」を管理）
  - `"player"` タグ … 衝突判定用
- **【確定】** プレイヤー状態（ノーマル / どんぐり / リンゴ / ふわふわ）は独自の状態機械として `src/` 内に切り出し、被弾・アイテム取得時に切替

### 敵

- **【確定】** イモムシ・ダンゴムシ・コバチ・ボス（ヌシイモ）はそれぞれ別エンティティファクトリ関数で生成
- **【確定】** 共通インターフェース: `踏まれた時の挙動` / `側面衝突時のダメージ` / `画面外退場` を持つ
- **【確定】** ダンゴムシの「丸まり → 蹴られて転がる」は内部状態を持たせて切替

### アイテム

- **【確定】** ドングリ / リンゴ / 葉っぱ / ベリー / きらきら花 / 金のヒナ / 種（プレイヤー攻撃弾）はファクトリ関数で生成
- **【確定】** プレイヤー衝突時のコールバックでスコア加算 / 状態遷移 / 残機加算を発火

### ボス

- **【確定】** ボス（ヌシイモ）は HP 3 を内部状態で持ち、攻撃パターン（左右移動 → ジャンプ叩きつけ → 隙）を有限状態機械で表現
- **【確定】** 踏みつけ時に HP -1、種ヒット時にカウンタ +1（5 で HP -1 換算）

## 物理・カメラ

- **【確定】** 重力は Kaboom.js の `setGravity()` を使う
- **【確定】** プレイヤー追従カメラは `setCamPos()` を毎フレーム更新
- **【確定】** 横スクロールは X 方向のみ追従、巻き戻しなし
- **【確定】** **縦スクロール採用面（1-4 空）** では Y 方向追従も有効化

## 入力

- **【確定】** Kaboom.js の `onKeyDown()` / `onKeyPress()` / `onKeyRelease()` で `controls.md` のキーを購読
- **【確定】** ジャンプの先行入力（短時間バッファ）は独自タイマーで実装

## アセットパイプライン

- **【確定】** 画像は **PNG**、アニメーションは**スプライトシート**（横並び）
- **【確定】** スプライト読み込みは `loadSprite("piyo", "/sprites/piyo.png", { sliceX: 8, anims: { idle: 0, walk: { from: 1, to: 4, loop: true }, ... } })` 形式
- **【確定】** ステージのレベルデータは **JSON**（タイル ID 配列または ASCII マップ）。Kaboom.js の `addLevel()` を使って配置
- **【確定】** アセットは **`public/` 配下**に配置し、Vite で静的アセットとしてそのまま配信
- **【仮】** プロトタイプは色付き矩形と簡易ドットで進め、本素材は後段で差し替え（`assets.md` 方針通り）

## 音響

- **【確定】** 初期リリースは無音のため、サウンド系 API は呼ばない
- **【確定】** 将来 BGM/SE を入れる際は `loadSound()` + `play()` で対応する想定

## 永続化

- **【確定】** ハイスコアは `localStorage` に **`piyo-game:highscore`** のキーで保存
- **【確定】** セーブ機能は持たない（仕様通り）

## ゲームループ

- **【確定】** Kaboom.js が `requestAnimationFrame` ベースのメインループを提供。各シーンは `onUpdate()` でフレームごとの処理を記述
- **【確定】** 制限時間カウントダウンや無敵タイマーは `wait()` / 独自カウンタを併用

## 未決事項

- レベルデータを JSON 直書きにするか、Tiled エディタの `.tmx → JSON` パイプラインを通すか（初期は手書き JSON で十分の見込み）
- スプライトシート以外のアニメーション形式（Aseprite JSON 出力等）を使うか
