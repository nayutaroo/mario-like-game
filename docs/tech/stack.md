# stack — 技術スタック

## 言語・ランタイム

- **【確定】** **TypeScript**（最新安定版、`strict: true` で運用）
- **【確定】** 実行環境はモダンブラウザ（Chrome / Firefox / Safari / Edge の最新版）。Node.js は開発時のビルド・パッケージ管理にのみ使用
- **【確定】** ランタイム想定: ES2022 以降の機能を許容（ターゲットは esnext、ビルド時にブラウザ向けに変換）

## ゲームライブラリ

- **【確定】** **Kaboom.js** を採用（軽量ゲームライブラリ、ポップなカートゥーン作品と相性が良い）
- **【確定】** ライブラリ提供の機能を活用する範囲：
  - シーン（Scene）切替
  - スプライト・アニメーション
  - AABB ベースの当たり判定
  - 重力・物理（`body()` / `area()` コンポーネント）
  - キーボード入力ハンドリング
  - スプライトシート読み込み
  - レベル定義（`addLevel()`）

### Kaboom.js / KAPLAY に関する注記

- **【確定】** Kaboom.js は 2024 年に元の保守チーム（Replit）が公式サポートを停止し、コミュニティが **KAPLAY** という名称でフォークして継続開発している（`kaplayjs.com`）
- **【確定】** API はほぼ完全互換。本作は当面 **`kaboom` パッケージ**（npm 上の最後の安定版）か **`kaplay` パッケージ**（後継）のいずれかを使用する
- **【仮】** 初期セットアップは `kaplay` を採用する。理由は (1) 保守継続性 (2) 型定義が継続更新されている。Kaboom.js 互換 API のためコードはほぼそのまま流用可能。
- **【未決】** 万一 KAPLAY の API が破壊的変更を入れた場合は、`kaboom@最終版` への切替手順を docs に追記する

## ビルドツール

- **【確定】** **Vite** を採用
  - 理由: TS のゼロ設定起動、HMR、`vite build` で静的アセット込みの最小バンドルが GitHub Pages にそのまま乗る
- **【確定】** 開発時は `vite` 単体で `localhost` 起動、本番は `vite build` で `dist/` に出力
- **【確定】** **`vite` バージョンは `>=8.0.5` を必須**（2026-04 公表の脆弱性 CVE-2026-39363 / -39364 / -39365 への対策）。詳細は [`security.md`](security.md) 参照

## パッケージマネージャ

- **【確定】** **npm**（追加ツールなしで動く）
- **【未決】** pnpm への切替は将来検討。本作の規模では恩恵が小さいため当面は npm

## リンタ / フォーマッタ

- **【確定】** **Biome** を採用（`biomejs/biome`）
  - 理由: lint + format を 1 ツールで完結、設定が小さい、Rust 製で高速
- **【確定】** TypeScript のチェックは `tsc --noEmit` を別途回す

## テスト

- **【確定】** 初期スコープではユニットテストを**導入しない**（小規模・個人制作・ゲームロジックは目視で動作確認しやすいため）
- **【未決】** 物理計算や状態遷移の純粋ロジックを切り出した時点で、`vitest` 導入を再検討

## デプロイ先

- **【確定】** **GitHub Pages**
- **【確定】** リポジトリ直下の `docs/` は仕様用に使うため、ビルド出力は `dist/` → `gh-pages` ブランチ（または GitHub Actions のartifact 経由）に配信する
- **詳細手順**: [`deploy.md`](deploy.md) 参照

## CI/CD

- **【確定】** **GitHub Actions** で `main` ブランチへのプッシュ時にビルド + GitHub Pages へのデプロイを実行
- **【未決】** PR ごとのプレビュー URL は当面不要（個人開発のため）

## ブラウザサポート

- **【確定】** Chrome / Firefox / Safari / Edge の最新 2 バージョン
- **【確定】** モバイル / タッチ対応は初期スコープ外
